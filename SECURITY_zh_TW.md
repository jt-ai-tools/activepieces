# 安全性

聯絡方式：security@activepieces.com

參考自 [https://supabase.com/.well-known/security.txt](https://supabase.com/.well-known/security.txt)

在 Activepieces.com，我們將系統的安全性視為首要任務。但無論我們在系統安全上投入多少努力，仍然可能存在漏洞。

如果您發現了漏洞，我們希望了解它，以便我們能盡快採取措施。我們請求您幫助我們更好地保護我們的客戶和系統。

## 超出範圍的漏洞：

- 無敏感操作頁面上的點擊劫持 (Clickjacking)。
- 未經身份驗證/登出/登入的跨站請求偽造 (CSRF)。
- 需要中間人攻擊 (MITM) 或對使用者裝置進行實體存取的攻擊。
- 任何可能導致我們服務中斷 (DoS) 的活動。
- 內容欺騙和文字注入問題，且未顯示攻擊向量/無法修改 HTML/CSS。
- 電子郵件欺騙。
- 缺少 DNSSEC、CAA、CSP 標頭。
- 非敏感 Cookie 缺少 Secure 或 HTTP only 標記。
- 失效連結。

## 請執行以下操作：

- 將您的發現透過電子郵件發送至 [security@activepieces.com](mailto:security@activepieces.com)。
- 請勿在我們的基礎設施或儀表板上執行自動掃描器。如果您希望這樣做，請聯絡我們，我們將為您設置沙箱環境。
- 請勿利用您發現的漏洞或問題，例如下載超過演示漏洞所需的數據，或刪除、修改他人的數據。
- 在問題解決之前，請勿向他人透露該問題。
- 請勿使用針對實體安全、社交工程、分散式阻斷服務、垃圾郵件或第三方應用程式的攻擊。
- 請提供足夠的資訊來重現問題，以便我們能盡快解決。通常，受影響系統的 IP 位址或 URL 以及漏洞描述就足夠了，但複雜的漏洞可能需要進一步說明。

## 我們的承諾：

- 我們將在 3 個工作日內回覆您的報告，提供我們對報告的評估以及預計解決日期。
- 如果您遵守上述指示，我們不會就報告對您採取任何法律行動。
- 我們將嚴格保密您的報告，未經您的許可不會將您的個人詳細資訊提供給第三方。
- 我們將讓您了解解決問題的進度。
- 在有關所報告問題的公開資訊中，我們將註明您為發現者（除非您另有要求）。
- 我們致力於盡快解決所有問題，並希望在問題解決後的最終發佈中發揮積極作用。
