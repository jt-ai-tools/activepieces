當我們開發新功能時，我們的行銷團隊會負責對外公告。身為工程師，我們需要清楚地溝通：

1. 該功能解決的問題
2. 對使用者的好處
3. 如何與我們的產品整合

### 交接給行銷團隊

GitHub 與 Linear 之間有整合功能，在 issue 關閉 5 分鐘後，會自動為行銷團隊建立一張票。
\
\
請確保以下事項：

- GitHub Pull Request 已連結至一個 issue。
- 該 Pull Request 必須具有以下標籤之一：**"Pieces"**、**"Polishing"** 或 **"Feature"**。
  - 如果未添加這些標籤，PR 將不會被合併。
  - 您也可以添加任何其他相關的標籤。
- GitHub issue 必須包含正確的範本（請參閱下方的「票券範本」）。

<Tip>
  額外加分：請附上一段影片向行銷團隊展示如何使用此功能，以便他們製作示範影片並進行正確的行銷。
</Tip>

票券範本：

```
### 此功能解決了什麼問題？

### 解釋功能如何運作
[在此插入影片連結]

### 目標受眾
企業版 / 所有人 

### 相關使用者情境
[在此插入 Pylon 票券或社群貼文]
```
