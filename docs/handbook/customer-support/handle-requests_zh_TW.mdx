作為支援工程師，你應該：

* 修復緊急問題（請參閱下方的定義）
* 為所有非緊急問題建立 Ticket。**(請勿在 Issue 中包含任何敏感資訊)**
* 持續更新客戶進度
* 撰寫清晰的 Ticket 描述
* 協助團隊安排工作優先順序
* 將問題分派給合適的人員

<Note>
我們的支援時間為 **紐約時間 (ET) 上午 8 點至下午 6 點**。在與客戶溝通預期回覆時間時，請記住這一點。
</Note>

### Ticket 欄位

處理支援 Ticket 時，請確保設定合適的狀態和優先順序，以利於 Ticket 管理和響應時間：

### 請求

### 類型 1：快速修復與緊急問題

* 了解問題及其緊急程度。
* 在 Linear 上建立一個帶有 "require attention" 標籤的 Ticket 並指派人員。

### 類型 2：複雜的技術問題

* 評估問題並確定其緊急程度。
* 務必為功能請求建立 Linear Issue，並將其發送給客戶。
* 在 Linear Issue 上留言註明預計完成時間。

### 類型 3：功能增強請求

* 務必為功能請求建立 Linear Issue，並將其發送給客戶。
* 評估請求並深入了解客戶試圖解決的問題，然後評估並建立新 Ticket，或將其附加到 Backlog 中的現有 Ticket。
* 將其加入我們的 Roadmap 並與團隊討論。

### 類型 4：商業案例
* 這些案例涉及購買新功能、計費或討論協議。
* 在 Pylon 上將 Team 更改為 "Success"。
* 標記客戶成功團隊 (Success Team) 的成員來處理。

<Tip>
新功能的狀態一律為 "Backlog"。請務必溝通我們將在未來的製作週期中討論並處理，以免客戶期望立即採取行動。
</Tip>

### 常見問題

<AccordionGroup>
  <Accordion title="如果我不了解功能或問題該怎麼辦？">
    如果您不了解功能或問題，請聯繫客戶進行澄清。在繼續之前充分掌握問題至關重要。您也可以諮詢您的團隊以獲得額外見解。
  </Accordion>

  <Accordion title="如何排定多個緊急問題的優先順序？">
    面對多個緊急問題時，評估每個問題對客戶和系統的影響。根據嚴重程度、受影響的用戶數量和潛在風險排定優先順序。與您的團隊溝通以確保優先順序達成一致。
  </Accordion>

  <Accordion title="如果遇到憤怒或辱罵的客戶該怎麼辦？">
    如果您遇到辱罵或粗魯的客戶，請將問題呈報給 Mohammad AbuAboud 或 Ashraf Samhouri。謹慎處理此類情況並確保客戶感到被傾聽，同時保持尊重和專業的態度，這點非常重要。
  </Accordion>

</AccordionGroup>
