秘密管理器允許您將外部秘密管理系統與 Activepieces 整合，從而實現集中式的憑證管理，並增強全域連接的安全性。

<Snippet file="enterprise-feature_zh_TW.mdx" />

## 優點

- **集中管理**：將所有憑證儲存在一個安全的位置
- **增強安全性**：憑證由專用的秘密管理系統管理
- **稽核與合規**：追蹤秘密的存取和變更
- **支援輪替**：輕鬆輪替憑證，無需更新流程 (flows)
- **存取控制**：使用您現有的秘密管理器存取政策

## 支援的供應商

- **[HashiCorp Vault](./hashicorp_zh_TW)** - 企業級秘密管理
- **[CyberArk Conjur](./cyberark-conjur_zh_TW)** - 具備主機型驗證的集中式秘密管理
- **[AWS Secrets Manager](./aws_zh_TW)** - AWS 上的託管秘密儲存
- **[1Password](./1password_zh_TW)** - 具備秘密自動化功能的個人與團隊密碼管理器

## 如何連接

1. 前往 **平台管理 → 安全 → 秘密管理器**
2. 選擇您想要連接的秘密管理器供應商
3. 按照供應商文件中的特定設定說明進行操作
4. 輸入必要的連接詳細資訊
5. 點擊 **Connect** 以測試並儲存連接

該連接將被加密並安全地儲存。您可以隨時點擊 **Remove** 按鈕中斷連接。

## 在全域連接中使用秘密管理器

連接後，您可以在建立或編輯全域連接時使用秘密管理器：

1. 設定需要憑證（如 API keys 或密碼）的全域連接時
2. 點擊輸入欄位旁邊的 **鑰匙圖示** (🔑)
3. 從下拉選單中選擇您的秘密管理器供應商
4. 輸入您的供應商所需的秘密路徑/識別碼 (secret path/identifier)（請參閱供應商特定文件）
5. 連接將在需要時自動從您的秘密管理器中檢索秘密

## 運作原理

當您在連接中使用秘密管理器時：

- 全域連接儲存的是對秘密的引用（而非實際憑證）
- 當流程執行時，Activepieces 會向您的秘密管理器進行驗證並檢索秘密
- 秘密是按需擷取的，且從不儲存在 Activepieces 中
- 如果秘密在您的秘密管理器中更新，流程將在快取過期後（最多 1 小時）使用新值，或者在手動清除快取後立即生效

## 快取

連線檢查和檢索到的秘密會加密快取在 Redis 中 **1 小時**，以減少延遲和供應商 API 負載。

若要強制重新整理（例如在輪替憑證或更新秘密後），平台管理員可以在秘密管理器設定頁面中清除快取

![清除秘密管理器快取](/resources/screenshots/clear-secret-manager-cache.png)

或透過呼叫以下端點來達成：

```
curl --request DELETE \
  --url https://<domain>/api/v1/secret-managers/cache \
  --header 'Authorization: Bearer <token>'
```

## 安全性考量

- **加密**：秘密管理器的身分驗證設定皆經過加密
- **存取控制**：使用您秘密管理器的存取政策來控制誰可以存取秘密
- **網路安全**：確保您的秘密管理器可以從您的 Activepieces 實例存取
- **憑證管理**：定期輪替秘密管理器的身分驗證憑證

## 故障排除

**連接失敗：**
- 驗證連接詳細資訊是否正確且可存取
- 檢查身分驗證憑證是否有效
- 確保 Activepieces 與您的秘密管理器之間的網路連線正常
- 查閱供應商特定的故障排除指南

**找不到秘密：**
- 驗證秘密路徑/名稱是否正確
- 檢查秘密是否存在於您的秘密管理器中
- 確保身分驗證憑證具有讀取秘密的權限

**權限不足：**
- 驗證身分驗證憑證是否具有必要的權限
- 檢查您秘密管理器的存取控制政策
- 查閱您秘密管理器中的稽核記錄以獲取詳細的錯誤資訊
