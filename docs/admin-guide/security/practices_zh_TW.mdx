## 外部系統憑證

**儲存憑證**

所有憑證均使用 256 位元加密金鑰儲存，且沒有可供使用者檢索它們的 API。它們僅在處理期間發送，之後引擎的存取權限會被撤銷。

**數據遮罩**

我們實施了強大的數據遮罩機制，第三方憑證或任何敏感資訊都會在日誌中系統性地被屏蔽，保證敏感資訊絕不被儲存或記錄。

**OAuth2**

與第三方的整合始終使用 OAuth2 進行，並在第三方支援允許的情況下使用有限數量的權限範圍 (scopes)。

## 存取與身份驗證 

**基於角色的存取控制 (RBAC)**

為了管理使用者存取，我們利用基於角色的存取控制 (RBAC)。團隊管理員為使用者分配角色，授予他們存取和與專案、資料夾和資源互動的特定權限。RBAC 允許細粒度的控制，使管理員能夠根據使用者角色定義和執行存取政策。

**單一登入 (SSO)**

實施單一登入 (SSO) 是我們安全性策略的關鍵組成部分。SSO 允許使用者使用一組憑證存取 Activepieces，從而簡化了使用者身份驗證。這不僅提高了使用者便利性，還透過減少與管理多個登入憑證相關的潛在攻擊面來加強安全性。

**稽核日誌**

我們維護全面的稽核日誌，以追蹤和監控 Activepieces 內的所有存取活動。這包括使用者互動、系統變更和其他相關事件。我們細緻的記錄有助於識別安全性威脅，並確保我們安全性措施的透明度和問責制。

**密碼政策強制執行**

使用者使用只有他們自己知道的密碼登入 Activepieces。Activepieces 強制執行密碼長度和複雜度標準。密碼不會被儲存；相反，資料庫中僅儲存密碼的安全雜湊值 (hash)。欲了解更多資訊。
