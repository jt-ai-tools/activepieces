# 使用 Pulumi 進行基礎設施即程式碼 (IaC)

Pulumi 是一款類似於 Terraform 或 CloudFormation 的 IaC 解決方案，讓您可以使用熱門的程式語言（例如我們將使用的 Typescript、C#、Go 等）來部署與管理您的基礎設施。

## 從 Pulumi Cloud 部署

如果您已經熟悉 Pulumi Cloud 並已[將其服務與您的 AWS 帳戶整合](https://www.pulumi.com/docs/pulumi-cloud/deployments/oidc/aws/#configuring-openid-connect-for-aws)，您可以使用下方的按鈕，只需點擊幾下即可部署 Activepieces。
該範本將部署 [Docker Hub](https://hub.docker.com/r/activepieces/activepieces) 上可用的最新 Activepieces 映像檔。

[![Deploy with Pulumi](https://get.pulumi.com/new/button.svg)](https://app.pulumi.com/new?template=https://github.com/activepieces/activepieces/tree/main/deploy/pulumi)

## 從本地環境部署

或者，如果您目前使用 S3 儲存桶來維護您的 Pulumi 狀態，您可以使用下方的範本，只需幾個指令即可直接從 Docker Hub 支撐並部署 Activepieces：

```bash
$ mkdir deploy-activepieces && cd deploy-activepieces 
$ pulumi new https://github.com/activepieces/activepieces/tree/main/deploy/pulumi
$ pulumi up
```

## 部署了什麼？

此範本設定為具有一定的靈活性，支援開發環境或更趨向生產環境的配置。
在 Stack 配置期間提供的選項將允許您選擇性地添加以下任何或所有各項：

* PostgreSQL RDS 執行個體。不選擇此項將使用本地 SQLite3 資料庫。
* 單節點 Redis 7 叢集。不選擇此項將意味著使用記憶體內快取 (in-memory cache)。
* 帶有 SSL 的完全符合標準的網域名稱。請注意，託管區域 (hosted zone) 必須已在 Route 53 中設定。
不選擇此項將意味著依賴使用應用程式負載平衡器 (ALB) 的 URL 透過標準 HTTP 存取您的 Activepieces 部署。

如需所有目前可用配置選項的完整列表，請查看 [GitHub 上的 Activepieces Pulumi 範本檔案](https://github.com/activepieces/activepieces/tree/main/deploy/pulumi/Pulumi.yaml)。

## 第一次設定 Pulumi

如果您是 Pulumi 新手，請繼續閱讀以設定您的本地開發環境，以便能夠部署 Activepieces。

### 先決條件

1. 確保您已安裝 [Node](https://nodejs.org/en/download) 和 [Pulumi](https://www.pulumi.com/docs/install/)。
2. [安裝並配置 AWS CLI](https://docs.aws.amazon.com/cli/latest/userguide/getting-started-install.html)。
3. [安裝並配置 Pulumi](https://www.pulumi.com/docs/clouds/aws/get-started/begin/)。
4. 建立一個 S3 儲存桶，我們將用它來維護我們為 Activepieces 部署所配置的各種服務的狀態：

```bash
aws s3api create-bucket --bucket pulumi-state --region us-east-1
```

<Tip>
    注意：[Pulumi 支援兩種不同的狀態管理方式](https://www.pulumi.com/docs/concepts/state/#deciding-on-a-state-backend)。
    如果您寧願使用 Pulumi Cloud 而不是 S3，請隨時跳過此步驟並在 Pulumi 註冊帳戶。
</Tip>

5. 登入 Pulumi 後端：

```bash
pulumi login s3://pulumi-state?region=us-east-1
```
6. 接下來，我們將使用 Activepieces Pulumi 部署範本來建立一個新專案、該專案中的一個 Stack，然後開始部署：

```bash
$ mkdir deploy-activepieces && cd deploy-activepieces 
$ pulumi new https://github.com/activepieces/activepieces/tree/main/deploy/pulumi
```

此步驟將提示您建立 Stack 並填充一系列配置選項，例如是否配置 PostgreSQL RDS 執行個體或使用 SQLite3。

<Tip>
    注意：選擇 Stack 名稱時，請使用具有描述性的名稱，例如 `activepieces-dev`、`ap-prod` 等。
    此解決方案使用 Stack 名稱作為建立的每個 AWS 服務的前綴。
    例如，您的 VPC 將被命名為 `<stack name>-vpc`。
</Tip>

7. 最後只需開始部署：

```bash
pulumi up
```

8. 提示時選擇 `yes`。部署完成後，您應該會看到一堆如下所示的 Pulumi 輸出變數：
```json
    _: {
        activePiecesUrl: "http://<alb name & id>.us-east-1.elb.amazonaws.com"
        activepiecesEnv: [
         . . . .
        ]
       }
```

這裡感興趣的配置值是 `activePiecesUrl`，因為它是您 Activepieces 部署的 URL。
如果您在 Stack 配置期間選擇添加完全符合標準的網域，它將顯示在這裡。
否則，您將看到應用程式負載平衡器的 URL。就這樣。

恭喜！您已成功將 Activepieces 部署到 AWS。

## 部署本地建置的 Activepieces Docker 映像檔

要部署本地建置的映像檔而不是使用官方 Docker Hub 映像檔，請繼續閱讀。

1. 在本地複製 Activepieces 儲存庫：

```bash
git clone https://github.com/activepieces/activepieces
```
2. 進入 `deploy/pulumi` 資料夾並安裝必要的 npm 套件：

```bash
cd deploy/pulumi && npm i
```
3. 此資料夾已經有兩個準備好的 Pulumi Stack 設定檔：`Pulumi.activepieces-dev.yaml` 和 `Pulumi.activepieces-prod.yaml`。
這些檔案已經包含了建立環境所需的所有配置。您可以隨意查看並根據需要編輯這些值。
讓我們繼續建立一個使用現有 `Pulumi.activepieces-dev.yaml` 檔案的開發 Stack 並開始部署。

```bash
pulumi stack init activepieces-dev && pulumi up
```

<Tip>
    注意：這裡需要對 `pulumi stack init` 指令使用 `activepieces-dev` 或 `activepieces-prod`，因為 Stack 名稱需要與資料夾中現有的 Stack 檔案名稱相符。
</Tip>

4. 在繼續之前，您現在應該會在終端機中看到所有將被配置的服務預覽。
一旦選擇 `yes`，系統將根據解決方案根目錄中的 `Dockerfile` 建置一個新映像檔（確保 Docker Desktop 正在執行），然後將其推送到新的 ECR，並為該 Stack 配置所有其他 AWS 服務。

恭喜！您已使用本地建置的 Docker 映像檔成功將 Activepieces 部署到 AWS。

## 自訂部署

正如您對任何 IaC 的期望，所有目前的配置選項以及與配置服務相關的低階細節都是完全可以自訂的。
例如，如果您希望 VPC 擁有三個可用區而不是兩個、使用較舊版本的 Redis 或為 PostgreSQL 添加一些額外的安全性群組規則，您可以在 `deploy` 資料夾內的 `index.ts` 檔案中更新所有這些內容及更多。

或者，您可能仍想部署官方 Activepieces Docker 映像檔而不是本地建置，但想更改某些服務。只需將 Stack 檔案中的 `deployLocalBuild` 配置選項設置為 `false` 並對 `index.ts` 檔案進行任何您想要的更改。

強烈建議在執行此操作之前查看 [Pulumi 文件](https://www.pulumi.com/docs/clouds/aws/)。
