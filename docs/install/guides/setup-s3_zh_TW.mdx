執行日誌和檔案預設儲存在資料庫中，但您稍後可以切換到 S3 而無需進行任何遷移；在大多數情況下，資料庫就足夠了。

建議先從資料庫開始，如有需要再切換到 S3。切換後，資料庫中過期的檔案將被刪除，所有內容都將儲存在 S3 中。無需手動遷移。

## 環境變數

| 變數 | 描述 | 預設值 | 範例 |
|----------|-------------|---------------|---------|
| `AP_FILE_STORAGE_LOCATION` | 儲存檔案的位置。若要使用 S3 儲存，請設定為 `S3`。 | `DB` | `S3` |
| `AP_S3_ACCESS_KEY_ID` | S3 相容儲存服務的存取金鑰 ID (Access Key ID)。如果 `AP_S3_USE_IRSA` 為 `true`，則不需要。 | 無 | |
| `AP_S3_SECRET_ACCESS_KEY` | S3 相容儲存服務的私密存取金鑰 (Secret Access Key)。如果 `AP_S3_USE_IRSA` 為 `true`，則不需要。 | 無 | |
| `AP_S3_BUCKET` | 用於檔案儲存的 S3 儲存貯體 (Bucket) 名稱。 | 無 | |
| `AP_S3_ENDPOINT` | S3 相容儲存服務的端點 (Endpoint) URL。如果已設定 `AWS_ENDPOINT_URL`，則不需要。 | 無 | `https://s3.amazonaws.com` |
| `AP_S3_REGION` | S3 儲存貯體所在的區域 (Region)。如果已設定 `AWS_REGION`，則不需要。 | 無 | `us-east-1` |
| `AP_S3_USE_SIGNED_URLS` | 用於將流量直接路由到 S3。如果 S3 儲存貯體是公開的，則應啟用此項。 | 無 | `true` |
| `AP_S3_USE_IRSA` | 使用 IAM Role for Service Accounts (IRSA) 連接至 S3。當為 `true` 時，不需要 `AP_S3_ACCESS_KEY_ID` 和 `AP_S3_SECRET_ACCESS_KEY`。 | 無 | `true` |
| `AP_MAX_FILE_SIZE_MB` | 允許上傳的最大檔案大小（以 MB 為單位），包括流程執行 (Flow Run) 的日誌。 | `10` | `10` |

<Tip>
**溫馨提示 #1**：如果 S3 儲存貯體支援簽名 URL (Signed URLs) 但需要透過公開網路存取，您可以將 `AP_S3_USE_SIGNED_URLS` 設定為 `true`，以將流量直接路由到 S3，並減輕 API 伺服器的負載。
</Tip>
