<Accordion title="決定沙箱模式" icon="code">

您應該根據您的使用案例以及是否為多租戶 (multi-tenant) 來決定部署的沙箱模式。以下是簡化的決定方式：

<Tip>
**小提示 #1**：對於多租戶設置，請使用 V8/Code 沙箱 (Sandboxing)。

它是安全的，且在 Kubernetes 中不需要具備權限的 (privileged) Docker 訪問。
通常不允許使用具備權限的 Docker 以防止 root 權限提升威脅。
</Tip>

<Tip>
**小提示 #2**：對於單租戶設置，請不使用沙箱 (No Sandboxing)。它速度更快，且不需要具備權限的 Docker 訪問。
</Tip>

<Snippet file="execution-mode_zh_TW.mdx" />

更多資訊請參考 [沙箱與 Worker](../architecture/workers_zh_TW#sandboxing)
</Accordion>

## 安裝

1. 將 `AP_EDITION` 環境變數設置為 `ee`。
2. 將 `AP_EXECUTION_MODE` 設置為 `UNSANDBOXED` 以外的任何值，請查看上述章節。
3. 執行實例後，前往 **Platform Admin -> Setup -> License Keys** 啟用授權金鑰。

![啟用授權金鑰設定](/resources/screenshots/activation-license-key-settings.png)

</Accordion>
<Accordion title="設置 HTTPS" icon="lock">
強烈建議設置 HTTPS，因為許多服務要求 Webhook URL 必須是安全的 (HTTPS)。這有助於防止潛在的錯誤。

要設置 SSL，您可以使用任何反向代理。如需逐步指南，請查看我們使用 [Nginx](../guides/setup-ssl_zh_TW) 的範例。
</Accordion>
