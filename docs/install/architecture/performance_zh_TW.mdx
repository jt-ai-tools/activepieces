## 效能

平均而言，Activepieces (自託管) 在單個實例（包括 PostgreSQL 和 Redis）上每秒可以處理 95 次流程執行，且延遲低於 300 毫秒。\
隨著實例資源的增加和/或增加更多實例，它可以進一步擴充。\

#### 基準測試

以下是重現此基準測試的方法：

1. 使用以下環境變數執行帶有 PostgreSQL 和 Redis 的 Activepieces：

```env
AP_EXECUTION_MODE=SANDBOX_CODE_ONLY
AP_FLOW_WORKER_CONCURRENCY=25
```

2. 建立一個包含 Catch Webhook 觸發器和 Webhook Return Response 動作的流程。
   

   ![簡單的 Webhook 流程](/resources/screenshots/simple-webhook-flow.png)
3. 從 Webhook 觸發器取得 Webhook URL 並在末尾加上 `/sync`。
4. 安裝基準測試工具，例如 [ab](https://httpd.apache.org/docs/2.4/programs/ab.html)：

```bash
sudo apt-get install apache2-utils
```

5. 執行基準測試：

```bash
ab -c 25 -n 5000 http://localhost:4200/api/v1/webhooks/GMtpNwDsy4mbJe3369yzy/sync
```

6. 檢查結果：

用於獲得上述結果的實例規格：

- 16GB RAM
- AMD Ryzen 7 8845HS (8 cores, 16 threads)
- Ubuntu 24.04 LTS

<Tip>
  這些基準測試是基於在 `SANDBOX_CODE_ONLY` 模式下執行 Activepieces。這**不**代表 Activepieces Cloud 的效能，Cloud 版本使用不同的沙箱機制來支援多租戶。欲瞭解更多資訊，請參閱 [沙箱機制](/install/architecture/workers_zh_TW#sandboxing)。
</Tip>
