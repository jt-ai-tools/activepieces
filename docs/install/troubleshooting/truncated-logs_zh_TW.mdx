## 總覽

如果您在流程執行日誌中看到 `(truncated)`，這表示日誌已超過允許的最大檔案大小。

## 運作原理

目前有一項限制，即單次執行的日誌檔案大小不能超過特定限制。當達到此限制時，引擎會自動移除 JSON 輸出中最大的鍵 (keys)，直到其大小符合允許的範圍。

<Note>
**這不會影響流程執行。** 即使日誌被截斷，您的流程仍會正常運作。
</Note>

## 已知限制

關於截斷的日誌有一個已知問題：

如果您**暫停**一個流程，然後**恢復**執行，且恢復的步驟引用了來自被截斷步驟的資料，流程將會失敗，因為被引用的資料已不在日誌中。

## 解決方案

您可以將 `AP_MAX_FILE_SIZE_MB` 環境變數增加到更高的值，以允許更大的日誌檔案：

```bash
AP_MAX_FILE_SIZE_MB=50
```

<Info>
**未來改進：** 計劃中的增強功能會將此限制從「每個日誌檔案」更改為「每個步驟」，這將提供對日誌大小更細緻的控制。此功能目前處於規劃階段。
</Info>
